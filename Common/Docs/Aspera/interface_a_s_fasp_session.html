- (void) addPath: (NSString *)  path

Adds a source path to the Fasp session. Paths to files and directories should be relative to the documents directory for uploads and relative to the docroot on the remote machine for downloads. A copy of the path is maintained by the session and no reference to the original path object is retained. (Note: in older versions of the SDK paths for uploads were required to be absolute. This is still possible however it is discouraged and paths relative to the documents directory are encouraged in their place. Relative paths are preferred to allow modifications to the documents directory, which in some cases such as upgrade happen outside of the embedding application's control).

- (void) addPaths: (id <NSFastEnumeration>)  paths

Add a list of paths to the session. paths should be an object such as NSArray or NSMutableArray that contains NSString instances and conforms to the NSFastEnumeration protocol

- (BOOL) addPathsFromDir: (NSString *)  dir
makeRelative: (BOOL)  makeRelative
error: (NSError **)  error 

Attempts to obtain the contents of the given directory, and adds each file contained therein using addPath: upon success. If makeRelative is set to YES, paths contained in the application's documents directory are added as relative paths (with respect to the application's documents directory), otherwise they are added as absolute paths. It is recommended to set makeRelative to YES (see addPath for more information). Upon failure the error parameter will be populated, if it is non-nil

- (BOOL) addPathToPersistentSessionFromSource: (NSString *)  src
destination: (NSString *)  dest 

Adds the given source/destination pair to the session. If the session is not a persistent session or if the session is not active and ready to accept paths NO will be returned. If the path is successfully added to the session YES will be returned. Note that persistent sessions can accept paths only while in the running state and after the connection to the remote host has been fully established - all attempts to add paths to a persistent session prior to this moment will fail. The ASTransferQueue delegate method queue:canAcceptPathsForPersistentSessionAtIndexPath: is triggered on the delegate (if implemented) once the session is ready to accept paths. The source path should be an absolute path and the destination path should be relative to the docroot on the destination server. See Persistent Sessions for more details

- (BOOL) cancelled

Returns true if the session has been cancelled. This method will only return true once cancellation has completed, i.e. if the session is still running this method will return false (but it may return true even while the session is still active). Note that cancelled sessions are marked as finished and that ASTransferQueue will not start a cancelled session that is waiting in the queue.

- (BOOL) cancelledOrStopped

Returns true if the session has been cancelled or stopped - equivalent to (cancelled || stopped)

- (NSString *) computedDestination

For uploads: If ASFaspSession::destination: is blank or nil and a non-nil account is set, account.remoteDir is returned. Otherwise ASFaspSession::destination: is returned.

For downloads: if ASFaspSession::destination: is blank or nil and a non-nil account is set, account.localDir will be returned, possibly modified. Otherwise, ASFaspSession::destination: is returned, possibly modified. These paths will be prepended with the documents directory if relative, or returned as is if absolute.

+ (ASFaspSession *) downloadSessionWithFaspURL: (ASFaspURL *)  url

Creates a fasp download session from the given ASFaspURL instance. Nil is returned if the url is itself nil. The following session properties are set (or not) from the URL:

  • account: not set
  • isUpload: NO
  • destination: not set
  • sources: the url's path is added using addPath:
  • cookie: set from the URL's cookie attribute
  • token: set from the URL's token attribute
  • createpath: set from the URL's createpath attribute
  • pathmtu: set from the URL's pathmtu attribute In addition, the session's sourceURL property will be set to the given ASFaspURL instance
- (BOOL) experiencedTokenRejection

Returns true if lastError is set to an error of the domain ASTransferErrorDomain and code ASTransferErrorTokenRejected

- (BOOL) failed

Returns true if the session failed. A session has failed if it is finished and has experienced transfer failure. Note that it is possible for this method and ASTransferSession::succeeded to return false at the same time, for instance for stopped or cancelled transfers.

- (BOOL) finished

Returns true if the session has finished. Finished sessions are never started by ASTransferQueue. A session that transitions from being active to being inactive is always marked as finished unless it has been stopped (in which case it is not marked as finished).

- (BOOL) hasError

Returns true if the session has an error set, in which case the error is retrievable with lastError. A session will have an error set if it has been cancelled, stopped or experiences transfer failure.

- (BOOL) stopped

Returns true if the session has been stopped. This method will only return true once the session has been fully stopped, i.e. if the session is still running this method will return false (but it may return true even while the session is still active). Note that stopped sessions are not marked as finished and that ASTransferQueue will start a stopped session that is waiting in the queue.

- (BOOL) succeeded

Returns true if the session succeeded. A session has succeeded if it completed transferring without being cancelled, stopped or experiencing failure. Note that it is possible for this method and ASTransferSession::failed to return false at the same time, for instance for stopped or cancelled transfers.

- (int) overwritePolicy
readwritenonatomicassign

Session overwrite policy (whether to overwrite files with the same name). Possible values are:

  • AS_OVERWRITE_NONE; use fasp default (currently DIFF)
  • AS_OVERWRITE_DIFF: overwrite if file is different from the source (note that this policy interacts with the resume policy)
  • AS_OVERWRITE_OLDER: overwrite the file if older than the source
  • AS_OVERWRITE_NEVER: never overwrite the file
  • AS_OVERWRITE_ALWAYS: always overwrite the file

The default is AS_OVERWRITE_NEVER

- (int) resumePolicy
readwritenonatomicassign

Session resume policy. Possible values are:

  • AS_RESUME_POLICY_NONE: never resume transfers
  • AS_RESUME_POLICY_ATTRS: resume transfers for those files for which the source and destination attributes match.
  • AS_RESUME_POLICY_FULL_CSUM: resume transfers for files for which the source and destination have the same name and an identical (full) MD5 checksum
  • AS_RESUME_POLICY_SPARSE_CSUM: resume transfers for files for which the source and destination have the same name and identical sparse MD5 checksums

The default is AS_RESUME_POLICY_ATTRS
