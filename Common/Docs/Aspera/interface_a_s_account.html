Detailed Description

Stores host address and credentials. Used by ASFaspSession and ASAccountController

#import <ASAccount.h>

Inherits NSManagedObject, and <ASTokenIssueServices>.

+ (ASAccount *) accountAtIndex: (NSInteger)  index

Returns the account at the given index, according to an enumeration of all existing ASAccount instances sorted first by host and then by user (both ascending alphabetically). The number of existing accounts can be retrieved using count. Be aware that the return value of the method for invalid indices is undefined - if there are 3 accounts and the account at index 5 is requested, nil or an arbitrary account may be returned.

+ (ASAccount *) accountFromFaspURL: (ASFaspURL *)  url

Returns an account that matches the host, user, ssh port and udp port given in the URL. Any password in the url is ignored. If no ssh or udp ports are given in the url they must match the default values. If no account is found matching all these parameters, an account with the given parameters is created. No other parameters (such as encryption cipher) are set, even if they are present in the url.

Returns an account with the given parameters. If an account with the same exact parameters exists, it will be returned. Otherwise, an account with the given parameters is created. Note that for an existing account to be returned it must match exactly - if even one parameter differs, a new account will be created.

+ (ASAccount *) accountWithHost: (NSString *)  host

forcePassword: (BOOL)  forcePassword

forceUdpPort: (BOOL)  forceUdpPort 

Returns an account with the given parameters. The method first searches for an existing account, then creates one if no existing account matches the given parameters. If one of the force parameters is set to YES, the corresponding parameter is excluded from the search and is instead set ("forced") on any existing account that matches the other criteria. For instance: [ASAccount accountWithHost:"host" user:@"user" password:@"pass" sshPort:22 udpPort:1618 forcePassword:NO forceSshPort:NO forceUdpPort:YES] will search for any existing account with the given host, user, password and ssh port - if any are found, their udp port will be set to 1618 and only then the account will be returned.

- (void) configureForDefaultTokenAuthentication

Configures the account for key-based account authentication with Aspera Enterprise, Connect and Faspex servers that is compatible with Aspera Connect. Such a configuration is often used in conjunction with FASP transfer token authentication. Note that this method is intended for very specific authentication scenarios; it is not a requirement to carry out token authentication. See Account vs Token Authentication for more information.

Password, sshPort and udpPort may be nil, 0 or 0, respectively, in which case those parameters will be excluded from the search (for instance, passing a nil password, an ssh port of 314 and a udp port of 0 will return an account with the given host, user and an ssh port of 314, if such exists, regardless of its password or udp port)

Saves all changes to the model that were made on the current thread. An exception is raised if a validation error occurs. In fact, saving any model saves all models that were modified on the same thread, so care should be taken not to leave unsaved or invalid models in memory.

- (void) setExtendedParametersFromFaspURL: (ASFaspURL *)  url

Sets extended parameters for the account from the given URL. At the moment this method sets only the encryption cipher: if the URL contains an 'enc' parameter that is set to 'aes128' or 'aes-128', the account's encryption cipher will be set to AS_CIPHER_AES128. Otherwise it will be set to AS_CIPHER_NONE.

- (void) setUserInfoObject: (id)  obj

Sets the given object for the given key on the userInfo dictionary of the account. Key/value pairs should never be set directly on the userInfo attribute, this method should be used instead. - (BOOL) validate: (NSError **)  err

Returns true if the account is valid, and false otherwise. If false is returned and err is non-nil, the variable err points to will be populated with the validation error(s). This method may be used to validate an account before it is saved.

- (BOOL) validateForDelete: (NSError **)  err

Returns true if the object is valid for deletion. An account is valid for deletion only if there are no transfer instances that reference it. If the account is not valid for deletion and err is non-nil, the variable it points to will be populated with the validation errors. This method may be used to validate an account before it is deleted.
