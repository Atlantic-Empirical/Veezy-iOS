ASTransferQueue Class Reference

Detailed Description

A transfer queue used to manage transfers. This class is the active component in the SDK responsible for initiating transfers from queued ASFaspSession objects

#import <ASTransferQueue.h>

Inherits NSObject, and <ASTransferDelegate>.

- (BOOL) cancelSessionAtIndex: (NSUInteger)  index

Cancel the session at the given index and return true. Canceling a session will interrupt the transfer, cause it to be deactivated and will prevent it from being automatically re-run by the queue. Cancelled sessions remain in the queue and can be re-run using runSessionAtIndex: (for most server configurations when re-run they will pick up from where they left off, though this depends the server configuration). If the index is out of bounds this method does nothing and returns false

- (BOOL) enqueueSession: (ASFaspSession *)  session

Add a session to the back of the queue and return true on success. If the session has already been added to the queue this method does nothing and returns false. Passing nil to this session will trigger an exception of type InvalidSessionError

This method will trigger the following delegate calls, in order:

  • willChangeQueue: (ASTransferQueueDelegate-p)
  • queue:didAddSessionAtIndexPath: (ASTransferQueueDelegate-p)
  • didChangeQueue: (ASTransferQueueDelegate-p)

- (BOOL) moveDownSessionAtIndex: (NSUInteger)  index

Moves the session at the given index one place backwards in the queue. If the session at the given index is already at the back of the queue (index equal to size of the queue, minus one) nothing is done. This method will return false if the given index is out of bounds, and true otherwise.

- (BOOL) moveSessionAtIndexToEnd: (NSUInteger)  index

Moves a session to the end of the queue. This method has no effect if the session at the given index is currently active.

- (BOOL) moveUpSessionAtIndex: (NSUInteger)  index

Moves the session at the given index one place forward in the queue. If the session at the given index is already at the front of the queue (index 0) nothing is done. This method will return false if the given index is out of bounds, and true otherwise.

- (BOOL) pauseSessionAtIndex: (NSUInteger)  index

Pause the session at the given index and return true. Pausing a session will leave it active, but no content will be transferred to the remote server while the session is paused. Since the session is not deactivated it will block the queue until it is resumed and finished, stopped or cancelled (or experiences an error). In order to interrupt a session and allow the queue to continue to other sessions use cancelSessionAtIndex:. If the index is out of bounds this method does nothing and returns false. Pausing an inactive session has no effect.

- (void) removeAllFinishedSessions

Removes all finished sessions from the queue

- (void) removeAllSessions

Removes all sessions from the queue

- (BOOL) removeSessionAtIndex: (NSUInteger)  index

Removes the session at the given index. If the given session is out of bounds nothing is done and false is returned. Otherwise upon successful removal, true is returned. The session object is destroyed by this call.

This method ignores session state - it will remove active as well as inactive sessions, and it is the caller's responsibility to make sure that a session is inactive before removing it

- (BOOL) resumeSessionAtIndex: (NSUInteger)  index

Resume the session at the given index and return true. Only sessions that have been previously paused using pauseSessionAtIndex: should be resumed using this method (in particular, calling this method on an inactive session will have no effect). When called on a previously paused session this method will cause the session to continue transferring to the remote host. If the index is out of bounds this method does nothing and returns false

- (BOOL) runSessionAtIndex: (NSUInteger)  index

Attempts to start the session at the given index. If the index is out of bounds, or if another session is already active, this method does nothing and returns false. Otherwise, it will start the session at the given index and return true. Note that this method is intended to be called on inactive sessions (in particular, sessions paused using pauseSessionAtIndex: remain active and should be resumed using resumeSessionAtIndex: rather than with this method).

Note that running one session while a different one is active will cause the active session to be stopped, the newly-run session to execute and, once it finishes, the originally running session will be run again.

- (ASFaspSession *) sessionAtIndex: (NSUInteger)  index

Returns the session at the given index. The session is refreshed before being returned (see refresh (ASFaspSession)). Nil will be returned if the given index is out of bounds.

- (NSArray *) sessionsForAccount: (ASAccount *)  account

Returns an array containing the sessions in the queue belonging to the given account. This method will save the account before returning the sessions, to ensure that the account is up to date (note that this requires the account to be in a valid state when invoking this method)

+ (ASTransferQueue *) sharedQueue

Returns the shared queue used by the application

- (void) start

Start the queue. This will set isRunning to true and start the queue background thread. ASTransferQueueDelegate::willStartQueue is called from the background thread and may be invoked only after this method returns. If the queue is already running, this call has no effect

- (void) stop

Stop the queue, cancelling any ongoing sessions. Once all sessions are cancelled, ASTransferQueueDelegate::didStopQueue will be invoked. If backgroundTaskDemarcationEnabled is set to YES, all tasks will be immediately marked as ended

- (void) stopActiveSessions

Stops the sole active session in the queue, if any

- (BOOL) stopSessionAtIndex: (NSUInteger)  index

Stop the session at the given index and return true. Stopping a session deactivates the session but leaves it eligible to be automatically re-run by the queue. This method is useful for interrupting a transfer, allowing the queue to pass to other transfers if the queue order has changed, all while knowing that the interrupted transfer will be eventually re-run without further intervention. If the index is out of bounds this method does nothing and returns false

A threshold (in seconds) to which the application's backgroundTimeRemaining is compared to by the queue at regular intervals. If the time remaining falls below this value all transfer activity ceases (any active sessions will be deactivated). Once the backgroundTimeRemaining increases past this value transfer activity resumes (any sessions that were previously de-activated by this mechanism will be run anew).

- (const NSString*) targetRateMode
readwritenonatomicretain

Determines how the target rate of a session is set before it is activated. If the target rate mode is set to ASTransferQueueTargetRateModeFromSettings the queue will force on the session the target rate obtained by passing targetBitRateForConnection: (ASConnectivitySettings) the current connection type. If the mode is set to ASTransferQueueTargetRateModeFromSession the queue will not modify the session's target rate. Finally, if the mode is set to ASTransferQueueTargetRateModeMinimum the queue will compare the session's target rate to that obtained from ASConnectivitySettings and use the lower amongst the two. The default is ASTransferQueueTargetRateModeFromSettings
